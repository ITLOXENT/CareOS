Audit Log Policy (Draft)

Objectives

- Capture security and clinical events.
- Maintain immutability and integrity.
- Support regulatory investigations.

Events to Log

- Authentication and authorization events
- Data access and changes
- Administrative actions
- System errors and security alerts

Retention

- Minimum 7 years for clinical records where required.
- Security events retained based on regulatory needs.

Access Controls

- Logs are write-once and access is least-privileged.
- Access is monitored and reviewed.
